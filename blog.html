Discuss the scope of var, let, and const

Var:

  • Scope of var is global or function level. Variable declared within a function has the function level scope otherwise global scope.
  • We can be re-declared multiple times using var key word and same variable name.

Let:

  • Scope of let is block level.
  • Variables declared within curly braces can not be accessed from outside the respective scopes.
  • We can not be re-declared multiple times using let keyword and same variable name.

Const:

  • Scope of const is also block level. Here block is confined to the curly braces ({ xxx }).
  • Value can not be updated once value is assigned to a const. But properties can be updated.
  • We can not be re-declared multiple times using const keyword and same variable name.

Tell us the use cases of null and undefined

Null:

  • Null means an empty or non-existent value. Null is assigned, and explicitly means nothing.
  • Null indicates the intentional absence of a value.

Undefined:

  • Undefined means a variable has been declared, but the value of that variable has not yet been defined.
  • Undefined indicate the unintentional absence of any object value.

What do you mean by REST API?

  • There are many kinds of APIs. One of them is REST API. REST stands for 'representational state transfer' and is a particular kind of architectural style that APIs considered “RESTful” are constrained by and “conform to”. REST APIs are a very common and important type of API that uses HTTP protocol for data transmission.
